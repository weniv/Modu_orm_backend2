부트스트랩과 차이: 부트스트랩은 모듈형으로 되어 있습니다. 모듈형으로 되어 있다는 얘기가 무슨 얘기냐면, 예를 들어 container의 경우 여러 CSS가 복합적으로 적용이 되어 있어요. 하나의 클래스에 여러 CSS가 붙어있는 통으로 적용되는 모듈형이라고 보시면 됩니다. 커스터마이징을 위해 css파일을 만들어야 합니다. 테일윈드는 하나의 CSS가 class로 선언이 되어 있습니다. 예를 들어, flex를 적용하고 싶다면 class='flex' font를 sky로 적용하고 싶다면 class='text-sky-500'으로 적용하면 됩니다.

장점: html파일만 보면 된다. Django + Tailwind, Django + React + Tailwind도 괜찮습니다.

단점: 부트스트랩보다 개발속도가 느립니다. 그래도 순수한 html, css보다는 월등히 빠릅니다. 그 많은 class 명을 외워야 합니다.

결론: 백엔드가 디자이너나 프론트엔드 개발자 없이 개발하기 위해서는 부트스트랩이고 백엔드가 디자이너는 있거나 상상하고 있는 UI가 있는데 프론트엔드 개발자가 없다면 테일윈드
